Notary App API Documentation

       API Documentation
=================

Base URL: http://localhost:3000



1. Add Candidate - POST (--MAIN--)


--------------------

Route: /api/addcandidate
Method: POST
Description: Add a new candidate and set default status to "contacted".
Parameters:
- uid (User ID): Number (required)

--------------------


2. Add Candidate - POST
------------------------
Route: /api/addcandidate
Method: POST
Description: Add a new candidate and set default status to "contacted".
Parameters:
- cid (Candidate ID): Number (required)
- uid (User ID): Number (required)
- candidateName: String (required)

3. Get All Candidates - GET
---------------------------
Route: /api/candidates
Method: GET
Description: Retrieve a list of all candidates along with their status, ID, name, and UID.
No Parameters.

4. Get Candidate Status - GET
------------------------------
Route: /api/candidates/:cid/status
Method: GET
Description: Retrieve the status, statusUpdatedAt, ID, and UID of a specific candidate.
Parameters:
- cid (Candidate ID): Number (required)

5. Update Candidate Status - PUT
---------------------------------
Route: /api/candidates/:cid/status
Method: PUT
Description: Update the status of a specific candidate. Requires JSON payload with "status" and "statusUpdatedAt" fields.
Parameters:
- cid (Candidate ID): Number (required)
JSON Payload:
{
  "status": String (required),
  "statusUpdatedAt": String (required, Format: YYYY-MM-DD HH:MM:SS)
}

6. Get Candidate Count by User - POST
-------------------------------------
Route: /api/candidates/count
Method: POST
Description: Retrieve the count of candidates for a specific user. Requires JSON payload with "uid".
Parameters:
JSON Payload:
{
  "uid": Number (required)
}

7. Get All Candidates with Status, ID, Name, and UID - GET
----------------------------------------------------------
Route: /api/candidates
Method: GET
Description: Retrieve a list of all candidates along with their status, ID, name, and UID.
No Parameters.
